# Security policy

## Reporting a vulnerability

Do not report suspected vulnerabilities or sensitive privacy issues through a public GitHub issue.

Use the repository’s **Private vulnerability reporting** flow:

<https://github.com/TheAnarchoX/redline-support/security/advisories/new>

Include the affected Redline version, browser and version, operating system, impact, safe reproduction steps, and any mitigation you have identified. Share only the minimum sensitive material required.

If GitHub says private vulnerability reporting is unavailable, open a public issue containing no vulnerability details and ask the maintainer for a private contact route.

## Supported versions

Security fixes are targeted at the current marketplace release. Reports for older releases are still useful, but you may be asked to confirm the behavior on the current version.

## Scope reminder

This repository contains only the public website, documentation, and support resources. It does not contain the proprietary Redline application source code.
