# Redline support

Use the [public support page](support.html) to choose the right route.

## Public issues are appropriate for

- Reproducible bugs demonstrated with non-sensitive public content.
- Documentation errors or accessibility problems.
- Feature suggestions described without private research material.

Search existing issues and read the troubleshooting guide before filing. Include the Redline version, browser and version, operating system, steps, expected result, actual result, and whether the behavior reproduces on a public page.

The [local environment summary builder](support.html#report-builder) can format the non-sensitive version and platform details. It does not submit or store the values. Review the generated text before pasting it into an issue.

## Never post publicly

- Redline JSON exports or browser profiles.
- Private URLs, selected passages, notes, tags, or dossier names.
- Credentials, tokens, cookies, or authentication material.
- Unredacted screenshots containing personal or confidential content.
- Security-vulnerability details.

Use the private route in [SECURITY.md](SECURITY.md) for sensitive reports.

## Data recovery

Redline has no server-side copy of a local vault. If data appears missing, do not uninstall the extension or clear more browser data. Confirm the browser profile and create a full JSON backup if the library still opens.

Without the original local vault or a Redline JSON backup, complete recovery is not possible.

## Scope

Support is best-effort and focuses on current Redline releases on supported desktop browsers. It cannot repair third-party websites, guarantee bibliography-style compliance, or investigate reports that require publishing confidential research.
