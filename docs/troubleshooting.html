Problem solving

Start with the page, profile, and setting.

Most issues come from protected pages, changed content, shortcut conflicts, browser-profile boundaries, or an option that needs review.

Before changing or reinstalling anything, export a JSON backup if you can still open the library. Then reproduce the issue on a non-sensitive public page.

The selection menu does not appear

  1. Confirm the selection menu is enabled in Redline settings.
  2. Reload the page if Redline was installed or updated after the page opened.
  3. Try a standard https:// article page. Browser settings, extension pages, PDF viewers, new-tab pages, and other protected surfaces can block extensions.
  4. Check whether another extension or the page itself replaces selection behavior.
  5. Use the Redline context-menu action as a fallback.

The panel or sidebar does not open

  1. Open Redline from the browser toolbar or extension menu.
  2. If a keyboard shortcut fails, check the browser’s extension-shortcut page for a conflict.
  3. In Firefox, use the toolbar or tab fallback if the sidebar does not appear as expected.
  4. Restart the browser after a Redline or browser update if the panel state appears stuck.

A saved highlight is missing from the page

Open the annotation in the library first. If the record is present, your data is still saved even though the visual highlight was not restored.

Restoration can fail when the page text changed, the same passage occurs several times, a client-side route rewrote the document, the content lives in an unsupported embedded viewer, or automatic restoration is disabled. Compare the saved quote with the current page and mark the record for review as needed.

Citation fields are incomplete

Keep the original page active, refresh its citation metadata, and review the result. Some sites publish very little structured metadata or use fields inconsistently. Edit the citation panel manually before exporting.

Unexpected duplicate or missing URLs

Redline normalizes URLs to help recognize the same source. Tracking-parameter and hash handling can affect whether two URLs are treated as equivalent. Review URL-normalization settings and compare the canonical parts of each saved record before merging or deleting anything.

An import is blocked

  1. Confirm the file is a Redline JSON export, not a CSV, Markdown, citation, or hand-edited file.
  2. Read the validation message and keep the original file unchanged.
  3. Check whether the file was truncated by a transfer or storage problem.
  4. Try a known-good earlier backup to separate a file problem from a current-vault problem.
  5. Do not post the export publicly. Report the validation message and a redacted description instead.

An export does not appear

  1. Open the browser’s downloads panel and check for a blocked or renamed download.
  2. Confirm Redline still has download permission.
  3. Check the browser’s configured download directory and available disk space.
  4. Try a smaller scope or JSON format to see whether the problem is specific to one export path.

Search misses something I can see online

Redline searches your saved local vault, not all text on the current page or the live web. Save the page or capture the passage first. Then search for the saved title, URL, domain, note, tag, dossier, or annotation text.

Data is missing after a profile change or uninstall

Redline’s storage belongs to the browser profile where it was created. Confirm the browser, operating-system account, and profile are the same ones used originally. Do not clear more data while investigating.

No server-side recovery exists.If the original local vault has been removed, a full Redline JSON backup is the only complete restore path.

Still stuck?

Collect your Redline version, browser and version, operating system, steps, expected result, actual result, and whether the issue occurs on a non-sensitive public page. Then use the appropriate support channel.

Protect your researchDo not attach a vault export, browser profile, private URL, credential, token, or unredacted screenshot to a public issue.
