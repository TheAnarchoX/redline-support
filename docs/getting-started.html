First-run guide

Build your first research trail.

Go from an open page to a saved source, a useful annotation, an organized dossier, and a portable backup.

Redline works beside the page while you read and in a full library when you want to review. Everything you save is kept in browser-managed local extension storage.

Before you beginRedline is intended for current desktop Chrome, Chromium, Microsoft Edge, Brave, and Firefox 142 or later. Use the official marketplace listing for your browser when release links are available on the homepage.

1. Open Redline

Use the Redline toolbar button or the browser’s extension menu. Chrome-family browsers open a side panel; Firefox uses a sidebar or tab fallback depending on browser behavior.

Pinning Redline to the toolbar makes capture easier, but it is not required. Redline cannot interact with browser-internal pages such as extension settings, the new-tab page, or other protected URLs.

2. Save a source

  1. Open a normal web page you want to keep.
  2. Open Redline and choose the page capture view.
  3. Review the title and URL, then add a note, tags, reading status, importance, or a dossier link.
  4. Choose Save source. The page becomes a searchable local record.

Redline saves a structured record—not a complete offline copy of the page. The live page can still change or disappear.

3. Capture an exact passage

  1. Select relevant text on a supported web page.
  2. Use Redline’s floating selection menu or the browser context menu.
  3. Classify the finding, add your own context or next step, and apply tags if useful.
  4. Save the annotation. Redline keeps the selected text, its source, and nearby anchoring context.

Common choices include highlight, claim, source, question, follow-up, quote, warning, definition, counterpoint, note, and to-do. The original wording remains available in your library even if a later page change prevents visual highlight restoration.

4. Connect the work

Create a dossier for a case, investigation, topic, or research question. Attach saved pages and annotations to keep the relevant trail together. Use tags for themes that should cut across several dossiers.

A useful rule of thumbA dossier answers “what body of work does this belong to?” A tag answers “what is this about or how should I classify it?”

5. Review and search

Open the library to browse pages, annotations, dossiers, tags, links, and source domains. Local search covers saved titles, URLs, notes, annotations, dossiers, tags, and domains; it does not search unsaved live web content.

/ or Ctrl/Cmd + KFocus library search
↑ ↓ then EnterMove through and open results
EscLeave search or close the current interaction
FiltersNarrow the local vault by record type and saved metadata

6. Make your first backup

  1. Open the library’s export area.
  2. Select the full library or archive scope.
  3. Choose JSON and start the export.
  4. Store the downloaded file somewhere you already back up.
JSON is the recovery format.Other exports are useful for reading, sharing, analysis, and citations, but they do not preserve every Redline field for a complete restore.
